Detailed Description

Lexical analyzer definition for the transpiler.

This file contains the tokenizing rules for the source language, written in the flex syntax. It is responsible for reading the source code and breaking it down into a stream of tokens that the parser (parser.y) can understand.

The lexer identifies:

  • Keywords (if, when, from, etc.)
  • Operators (||, &&, +=, +, etc.)
  • Standard library function names (printf, malloc, sin, etc.)
  • Literals (integers, floating-point numbers, strings, characters, booleans)
  • Identifiers (variable names, user-defined function names)

For each recognized pattern (lexeme), it returns a corresponding token type to the parser. For literals and identifiers, it also passes the actual value through the yylval union.
