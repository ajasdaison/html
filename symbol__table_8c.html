Detailed Description

Implementation of a simple symbol table for a compiler or interpreter.

This file contains the data structures and functions to manage symbols, such as variables and functions, during the compilation process. The symbol table is implemented as a single linked list and supports nested scopes. When a new scope is entered, current_scope is incremented. When the scope is exited, all symbols declared within that scope are removed from the table, ensuring correct symbol resolution according to scoping rules.

void enter_scope ( )

Enters a new scope by incrementing the current scope counter.

This function should be called whenever a new block that introduces a new scope (like a function body or a compound statement) is entered in the source code.

◆ exit_scope()

void exit_scope ( )

Exits the current scope and removes all symbols declared within it.

This function iterates through the symbol table and deallocates any symbols whose scope level matches the current scope. It should be called when a scope block is exited. It then decrements the scope counter.

Inserts a new symbol into the symbol table.

A new Symbol is allocated and populated with the given name, type, and kind. The symbol's scope is set to the current_scope. The new symbol is inserted at the head of the symbol table's linked list.

◆ lookup_symbol()

Symbol * lookup_symbol ( char * name)

Looks up a symbol by name in the symbol table.

This function searches for a symbol with the given name. If multiple symbols with the same name exist in different scopes, it returns the one in the innermost (highest-numbered) scope, correctly modeling scope resolution.

Parameters
nameThe name of the symbol to look up.
Returns
A pointer to the found Symbol, or NULL if the symbol is not found.
