Detailed Description

This file defines the structures and functions for creating and manipulating the Abstract Syntax Tree (AST).

The AST is a tree representation of the source code, where each node represents a construct occurring in the code. This file provides the necessary enums for node types and data types, the main AST_NODE structure, and function prototypes for creating different types of AST nodes.

◆ NODETYPE

enum NODETYPE

Defines the type of each node in the Abstract Syntax Tree.

This enumeration is used to differentiate between various constructs in the source language, such as literals, operators, keywords, and function calls. Each member of this enum corresponds to a specific type of node that can be created in the AST.

Enumerator
int_literals 

Node representing an integer literal.

This node type is used for integer values found in the source code, such as 10, 42, etc. The actual integer value is stored in the idval field of the AST_NODE structure.

float_literals 

Node representing a floating-point literal.

This node type is used for floating-point values, such as 3.14, 0.5, etc. The value is stored in the fval field of the AST_NODE structure.

identifiers 

Node representing an identifier.

Identifiers are names used to identify variables, functions, etc. The name of the identifier is stored as a string in the str field of the AST_NODE structure.

string 

Node representing a string literal.

This is for string literals enclosed in double quotes, like "hello". The string content is stored in the str field of the AST_NODE structure.

characters 

Node representing a character literal.

This is for single character literals, like 'a'. The character value is stored in the cval field of the AST_NODE structure.

boolean 

Node representing a boolean literal.

This is for boolean values, i.e., true or false. The boolean value is stored in the bval field of the AST_NODE structure.

unary_operator 

Node representing a unary operator.

This is for operators that act on a single operand, such as - (negation) or ! (logical not). The operator symbol is stored in the operator field, and the operand is the left child.

binary_operator 

Node representing a binary operator.

This is for operators that act on two operands, like +, -, *, /. The operator symbol is in the operator field, and the operands are the left and right children.

compound_node 

Node representing a compound statement or a block of code.

This node groups a sequence of statements together, often enclosed in braces {...}. The left child points to the first statement, and subsequent statements are linked via the next pointer.

break_node 

Node representing a 'break' statement.

This node is used to terminate a loop or a switch statement prematurely.

skip_node 

Node representing a 'skip' or empty statement.

This node can represent an intentional no-op.

jumpTo_node 

Node representing a 'goto' or jump statement.

This node is used for unconditional jumps to a label. The target label is stored in the label child.

continue_node 

Node representing a 'continue' statement.

This node is used to skip the rest of the current loop iteration and proceed to the next one.

label_node 

Node representing a label.

This node marks a position in the code that can be jumped to by a jumpTo statement. The label name is stored in the label child.

assign_node 

Node representing an assignment operation.

This node handles assignments like x = 5. The left child is the identifier being assigned to, and the right child is the expression being assigned. The assignment operator (e.g., =, +=) is stored in the operator field.

if_statement 

Node for an 'if' statement.

Represents an if-else construct. The left child is the condition, the right child is the if block (then-statement), and the next pointer can link to an else or else if block. when_statement 

Node for a 'when' statement (likely a switch-like or conditional block).

This node represents a conditional statement similar to a simple if. The left child holds the condition, and the right child holds the statement to execute.

do_when_statement 

Node for a 'do-when' statement (likely a do-while loop).

Represents a loop that executes the statement block first, then checks the condition. The left child is the statement block, and the right child is the loop condition.

Node for a 'from' statement (likely a for loop).

choice_statement 

Node for a 'choice' statement (likely a switch statement).

This is the main container for a switch-like construct. The left child is the expression to be evaluated. The right child points to the list of case statements.

case_statement 

Node for a 'case' statement within a 'choice'.

Represents a single case in a switch statement. The left child is the value to match, and the right child is the statement to execute. Subsequent cases are linked via the next pointer.

stdio_function 

Node for a standard I/O function call (from stdio.h).

Used for functions like printf, scanf. The specific function is identified by an integer code, and the arguments are linked as children.

string_function 

Node for a string manipulation function call (from string.h).

Used for functions like strcpy, strlen. The specific function is identified by an integer code, and the arguments are linked as children.

math_function 

Node for a mathematical function call (from math.h).

Used for functions like sin, cos, sqrt. The specific function is identified by an integer code, and the arguments are linked as children.

stdlib_function 

Node for a standard library function call (from stdlib.h).

Used for functions like malloc, exit. The specific function is identified by an integer code, and the arguments are linked as children.
